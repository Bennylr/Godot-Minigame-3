## Devlog
In the past when I usually code, I always write one line of code at a time to make sure I'm writing out the code perfectly. I usually use print statements as I go to make sure those lines are actually doing what I want before I move on. But sometimes I do write everything at once, though not the whole project. I only do this when I'm in a rush or I simply just want the project done quickly. Thinking about methodology, I feel like it's honestly hurting my process due to the fact that I tend to do my work much later than I plan to. I am currently trying to work on a plan to fix this issue by setting some time to do my work and also eliminating distractions like my phone or even my bed.
